Introduction to AJAX

AJAX (Asynchronous JavaScript and XML) allows web pages to be updated asynchronously by exchanging data with a server. Here are key concepts and methods I've learned:

AJAX Methods and Concepts

XMLHttpRequest Object

const xhr = new XMLHttpRequest(); xhr.open('GET', 'url', true); xhr.onload = function() { if (xhr.status === 200) { console.log(xhr.responseText); } }; xhr.send();

The traditional way to make AJAX requests.

Fetch API

fetch('url') .then(response => response.json()) .then(data => console.log(data)) .catch(error => console.error('Error:', error));

Modern promise-based API for making HTTP requests.

Async/Await with Fetch

async function getData() { try { const response = await fetch('url'); const data = await response.json(); console.log(data); } catch (error) { console.error('Error:', error); } }

Cleaner syntax for working with promises.

POST Request

fetch('url', { method: 'POST', headers: { 'Content-Type': 'application/json', }, body: JSON.stringify(data), }) .then(response => response.json()) .then(data => console.log(data));

Sending data to the server.

Handling JSON

// Parsing JSON const obj = JSON.parse(jsonString); // Stringifying to JSON const jsonString = JSON.stringify(obj);

Working with JSON data format.

Error Handling

fetch('url') .then(response => { if (!response.ok) { throw new Error('Network response was not ok'); } return response.json(); }) .catch(error => console.error('Error:', error));

Properly handling errors in AJAX requests.

Cross-Origin Requests (CORS)

// Server-side solution Access-Control-Allow-Origin: * // Or with credentials Access-Control-Allow-Credentials: true

Understanding and handling cross-origin resource sharing.
