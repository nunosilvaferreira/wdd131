Introduction to Git and GitHub

Git is a distributed version control system, and GitHub is a platform for hosting Git repositories. Here are essential Git commands I've learned for terminal usage:

Essential Git Commands

Initializing a Repository

git init

Creates a new Git repository in the current directory.

Cloning a Repository

git clone https://github.com/user/repo.git

Creates a local copy of a remote repository.

Checking Status

git status

Shows the current state of the working directory and staging area.

Adding Changes to Staging

git add filename

Adds a specific file to the staging area.

git add .

Adds all changed files to the staging area.

Committing Changes

git commit -m "Commit message"

Records changes to the repository with a descriptive message.

Viewing Commit History

git log

Shows the commit history.

git log --oneline

Shows a condensed version of the commit history.

Creating a Branch

git branch branch-name

Creates a new branch.

Switching Branches

git checkout branch-name

Switches to the specified branch.

git checkout -b new-branch

Creates and switches to a new branch in one command.

Merging Branches

git merge branch-name

Merges the specified branch into the current branch.

Remote Repositories

git remote add origin https://github.com/user/repo.git

Adds a remote repository.

git remote -v

Lists all remote repositories.

Pushing Changes

git push origin branch-name

Uploads local branch commits to the remote repository.

Pulling Changes

git pull origin branch-name

Downloads changes from the remote repository and merges them.

Stashing Changes

git stash

Temporarily saves changes that you don't want to commit immediately.

git stash pop

Restores the most recently stashed changes.

Reverting Changes

git reset --hard HEAD

Discards all local changes in the working directory.

git revert commit-hash

Creates a new commit that undoes the changes from a previous commit.
